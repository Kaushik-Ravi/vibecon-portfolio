Case Study 04

Toilet Watch

Citizen-powered sanitation intelligence with AI triage and accountability loops.

Role: Product architecture and full-stack delivery

Core stack: FastAPI, Gemini, Supabase, WAHA, Leaflet, Resend

Status: Public deployment

Problem

Public toilet hygiene reporting is usually fragmented, slow, and difficult for citizens to use consistently. Agencies often receive delayed information without clear triage or resolution traceability.

Approach

Toilet Watch uses a zero-install WhatsApp flow for citizen reports, then applies AI scoring and venue enrichment to route actionable alerts to responsible agencies. A public map and SLA escalation loop make performance visible and hard to ignore.

  • WhatsApp-first report ingestion with structured context capture
  • AI severity scoring and issue labeling from on-site photos
  • Google Places API enrichment for shop name, website, and contact metadata

Key Constraints and Fixes

  • Prototype uses WAHA for WhatsApp automation and is planned to migrate to Meta WhatsApp API
  • Photo-only reports lacked venue context until Places API enrichment was added
  • Time-window anti-spoof reporting controls are in progress and not fully deployed yet

Key Technical Decisions

  • Used WAHA as a prototyping bridge to validate WhatsApp-first behavior quickly
  • Added Places API enrichment to convert image-only reports into actionable venue records
  • Retained explicit pending status for time-lock anti-spoof controls to avoid false readiness claims

Outcomes

  • Lowered friction for citizens to report sanitation issues in seconds
  • Improved agency response workflow with direct email action controls
  • Introduced transparent public monitoring for sanitation performance
